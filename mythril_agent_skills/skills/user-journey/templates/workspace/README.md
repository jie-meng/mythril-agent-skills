## How to view

**Double-click `index.html`** — opens in your default browser. Works offline.

The page opens with a **left sidebar** listing every screen and a top **Canvas / Prototype** view switcher:

- **Canvas** (default): a single Miro-style canvas with every screen laid out in columns by stage, connected by arrows. Pan and zoom freely. A minimap in the bottom-right lets you click-to-pan.
- **Prototype**: one screen at actual size. Elements with outgoing arrows are clickable hotspots (dashed blue ring). Click them to walk the user flow. Breadcrumb at the top tracks your path; press `Backspace` to step back.

If the page is blank, or shows only the original 3 seed screens after edits, the inline copy of `journey.json` inside `index.html` has gone stale (browsers block `file://` fetches, so the page falls back to whatever was inlined at bootstrap). Two options:

```bash
python3 preview.py
```

Opens [http://localhost:8765](http://localhost:8765) automatically (uses real `journey.json` over HTTP — always current). Press `Ctrl+C` to stop. Requires only Python 3.10+.

Or, re-sync the inline copies so double-click works again:

```bash
python3 SKILL_PATH/scripts/sync_index_html.py .
```

(Your AI assistant runs this automatically when invoked through the `user-journey` skill; you only need it after a hand-edit.)

## Editing the journey

Ask your AI assistant — invoke the `user-journey` skill and give it this directory path. It reads all three files and continues with full context.

If you edit `JOURNEY.md`, `journey.json`, or `DESIGN.md` by hand, run all three gates:

```bash
python3 SKILL_PATH/scripts/validate_sync.py    .
python3 SKILL_PATH/scripts/validate_screens.py .
python3 SKILL_PATH/scripts/sync_index_html.py  .
```

The first two confirm the journey stays consistent (stages ↔ md ↔ screens ↔ arrows). The third re-injects the current `journey.json` and `DESIGN.md` tokens into `index.html` so the double-click path keeps showing live content. `validate_sync.py` also reports inline-JSON drift as an error, so a missed `sync_index_html.py` gets caught on the next validation pass.

## Changing visual style

Edit `DESIGN.md` directly. Via `preview.py` the renderer re-reads it on every page load. For the double-click path, run `python3 SKILL_PATH/scripts/sync_index_html.py .` to refresh the inline copy first. To swap presets, copy a different preset from `SKILL_PATH/templates/design-styles/` over `DESIGN.md`, then re-run the sync script.

## How to view

**Double-click `index.html`** — opens in your default browser. Works offline.

The page opens with a **left sidebar** listing every screen and a top **Canvas / Prototype** view switcher:

- **Canvas** (default): a single Miro-style canvas with every screen laid out in columns by stage, connected by arrows. Pan and zoom freely. A minimap in the bottom-right lets you click-to-pan.
- **Prototype**: one screen at actual size. Elements with outgoing arrows are clickable hotspots (dashed blue ring). Click them to walk the user flow. Breadcrumb at the top tracks your path; press `Backspace` to step back.

If the page is blank, your browser is blocking `file://` fetches:

```bash
python3 preview.py
```

Opens [http://localhost:8765](http://localhost:8765) automatically. Press `Ctrl+C` to stop. Requires only Python 3.10+.

## Shortcuts

### Both views

| Action | Shortcut |
|---|---|
| Switch to Canvas | `V` |
| Switch to Prototype | `P` |
| Show shortcuts | `H`, or click **?** in the topbar |

### Canvas mode

| Action | Shortcut |
|---|---|
| Pan | Drag empty canvas, or hold `Space` + drag |
| Zoom in / out | `+` / `−`, or `Cmd/Ctrl` + mouse wheel, or pinch on trackpad |
| Reset to 100% | `0` |
| Fit all to screen | `F`, or click **Fit** in the topbar |
| Focus a screen | Double-click it |
| Pan via minimap | Click anywhere on the bottom-right minimap |
| Nudge view | `←` / `→` / `↑` / `↓` |

### Prototype mode

| Action | Shortcut |
|---|---|
| Follow an arrow | Click any element with a dashed blue ring |
| Follow the default arrow | `Enter` |
| Next / previous screen | `J` / `K` |
| Go back | `Backspace`, or click **← Back** in the breadcrumb |

## Editing the journey

Ask your AI assistant — invoke the `user-journey` skill and give it this directory path. It reads all three files and continues with full context.

If you edit `JOURNEY.md` or `journey.json` by hand, run BOTH validators:

```bash
python3 SKILL_PATH/scripts/validate_sync.py    .
python3 SKILL_PATH/scripts/validate_screens.py .
```

to confirm the journey stays consistent (stages ↔ md ↔ screens ↔ arrows).

## Changing visual style

Edit `DESIGN.md` directly. The renderer reads it on every page load. To swap to a different preset, copy a different preset from `SKILL_PATH/templates/design-styles/` over `DESIGN.md` and refresh the browser.

## Sharing

Zip the entire directory and send it. Recipient unzips and double-clicks `index.html`. No install needed.

For a public link, commit to a git repo with GitHub Pages enabled — `index.html` works as-is.
